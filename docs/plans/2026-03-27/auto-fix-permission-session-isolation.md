# Auto-Fix: Permission prompts pollute across sessions

> **For Claude:** This plan was generated by auto-fix skill. Execute autonomously.

**Task:** Permission and question prompts from one session show up in the iOS app even when viewing a different session.

**Root Cause / Approach:** `permission_handler.broadcast()` sends to ALL WebSocket clients without checking which session the iOS app is viewing. The iOS-side filter exists but is bypassed when `session_id` is empty (happens for pending sessions). Fix: add server-side filtering in `http_server.py` — if a permission/question comes from a non-viewed session, return terminal fallback immediately instead of broadcasting.
