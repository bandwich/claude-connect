# Auto-Fix: Fix Session Switching Navigation

> **For Claude:** This plan was generated by auto-fix skill. Execute autonomously.

**Task:** Fix delayed session switching - selecting a new Claude session doesn't work until navigating back

**Root Cause:** `SessionsListView` uses `navigationDestination(isPresented:)` with a separate `selectedSession` state, causing timing issues. When tapping a new session, SwiftUI may evaluate the navigation destination before state updates are fully applied, or may not properly trigger `onAppear` for the new session when navigation state is already "active". This results in the session switch action being delayed until the next navigation event.

**Symptom:** When opening some sessions, the app shows messages from Claude that don't exist in the actual session file.

**Root Causes:**
