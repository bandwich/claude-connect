# Auto-Fix: Multiline Expanding Text Input

> **For Claude:** This plan was generated by auto-fix skill. Execute autonomously.

**Task:** Make the iOS text input expand vertically as the user types multiple lines.

**Root Cause:** `UITextView` inside `UIViewRepresentable` doesn't reliably propagate intrinsic content size changes to SwiftUI's layout. The fix is to explicitly calculate height via `sizeThatFits` and report it back through a binding so SwiftUI can update the frame.

---

## Changes

### 1. CommandTextField — add dynamic height reporting

**Files:**
- Modify: `ios-voice-app/ClaudeVoice/ClaudeVoice/Views/CommandTextField.swift`

**Steps:**
1. Add `@Binding var dynamicHeight: CGFloat` parameter
2. Calculate height in coordinator's `textViewDidChange` using `sizeThatFits`, capped at max ~120pt
3. Report initial height in `makeUIView`
4. Also recalculate in `updateUIView` when text changes externally (e.g., cleared after send)

### 2. SessionView — wire up height binding

**Files:**
- Modify: `ios-voice-app/ClaudeVoice/ClaudeVoice/Views/SessionView.swift`

**Steps:**
1. Add `@State private var textFieldHeight: CGFloat = 36`
2. Pass binding to `CommandTextField`
3. Apply `.frame(height: textFieldHeight)` instead of `.frame(minHeight: 36)`
